1. В якому статусі можуть перебувати файли при роботі з Git?
Всього статусів є три: не змінено (Unmodified), змінено (Modified), підготовлено (Staged).
2. Для чого використовується команда add?
Команда "add" використовується для того, щоб додати файл до коміту.
3. Для чого використовується команда status?
Ця команда використовується, щоб подивитися статус всіх файлів і самого репозитора.
4. Для чого використовується команда commit?
Команда "commit" використовується для того, щоб робити коміт змін.
5. Для чого використовується команда init?
Команда "init" створює git репозиторії, а також директорію .git в поточній директорії.
6. Що необхідно писати в коментарях комітів?
Пояснити зміни більш детально, якщо необхідно, виконати переноси рядків на 72 символах. 
Роз'ясніти, яку тему постановляє даний коміт.
Приділити багато уваги тому, що ми внесли у ці зміни, але не тому, як саме ми це зробили.
7. Для чого використовується команда checkout?
Команда "checkout" використовується для того, щоб переміщатися між вітками.
8. Для чого використовується команда reset?
Ця команда використовується для того, щоб взяти поточну гілку та перенести її в інше місце.
Команда "reset" має три основні форми, що відповідають аргументам командного рядка --soft, --mixed, --hard.
9. Для чого використовується команда log? 
Ця команда використовується для отримання останніх комітів графічної бібліотеки з відкритим вихідним входом.
10. Для чого використовується команда status?
Команда "status" використовується, щоб подивитися статус всіх файлів і самого репозитора.
11. Як відновити кілька файлів з попередньо створеного коміта?
Щоб відновити кілька файлів з попередньо створеного коміта, потрібно використати команду "git ls-files -d"
12. Як перейти до попереднього коміта по імені?
Щоб перейти до попереднього коміта по імені, використовуємо команду "git checkout <ID коміту>".
13. Як створити нову гілку?
Щоб створити нову гілку, достатньо виконати команду "git branch".
14. Як перейти на нову гілку?
Перейти на нову гілку, допоможе команда "git checkout <назва гілки>".
15. Як перейти на основну гілку?
Щоб перейти на основну гілку, використовується команда "git checkout master".
16. Як виконати злиття гілок?
Щоб виконати злиття гілок, спочатку переходимо на основну гілку, а потім використовуємо команду "git merge".
